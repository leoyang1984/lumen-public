# 运行时工作流最佳实践

## 1. 白板只写任务，不写排程

推荐把白板理解成“这件事怎么做”：

```text
输入 -> 提示词 -> 输出
```

把运行时面板理解成“什么时候自动做”：

```text
运行时间
是否启用
结果保存到哪里
```

这样用户只需要学习一种排程入口：运行时面板。

## 2. 先决定结果是“最新状态”还是“历史记录”

如果结果只是一个会不断更新的状态页，用“覆盖最新”：

```text
Projects/项目名/项目健康检查-最新.md
```

如果每次运行都应该留下记录，用“按日期保存历史”：

```text
Daily/总结/{{date}}-每日总结.md
```

简单判断：

```text
需要回看历史 -> 路径里放 {{date}}
只看当前结果 -> 路径里不放 {{date}}
```

## 3. 每日总结要明确区分“完成”和“待跟进”

Daily 内容通常会同时包含完成事项、想法、计划、待确认事项。提示词里建议写清楚：

```md
只有明确标记为已完成、已确认、已生成、已修复、已验证、已观察到结果的内容，才能归入“今日完成”。

包含“需要、准备、计划、想、待确认、待验证、继续、后续”的内容，应归入“待办 / 跟进”。

不要把“等待验证的事项”改写成已经完成。
```

## 4. 每日任务优先使用动态当天输入

如果日记命名为 `Daily/YYYY-MM-DD.md`，白板里放一个输入节点：

```md
#lumen/input
folder: Daily
date: today
```

也可以写成：

```md
#lumen/input
path: Daily/{{date:YYYY-MM-DD}}.md
```

这样同一张白板每天会自动读取当天笔记，不需要每天改白板。

## 5. 常见周期任务模板

### 每日日志总结

运行时间：

```text
每天 21:00
```

保存路径：

```text
Daily/总结/{{date}}-每日总结.md
```

提示词重点：

```text
今日完成
待办 / 跟进
灵感
状态与复盘
```

### 今日待办提取

运行时间：

```text
每天 08:30
```

保存路径：

```text
Daily/待办/{{date}}-今日待办.md
```

提示词重点：

```text
提取待办、承诺、等待、需要跟进的事项
保留 checkbox 格式
不要包含已经完成的事项
```

### 项目健康检查

输入节点：

```md
#lumen/input
path: Projects/项目名/项目状态.md
```

保存路径：

```text
Projects/项目名/项目健康检查-最新.md
```

提示词重点：

```text
当前状态
近期进展
风险
下一步行动
需要注意的阻塞
```

### Inbox 每日整理

输入节点：

```md
#lumen/input
path: Inbox.md
```

保存路径：

```text
Inbox/整理/{{date}}-Inbox整理.md
```

提示词重点：

```text
归类信息
提取任务
识别需要归档的材料
列出需要用户决策的事项
```

### 运行时日志复盘

输入节点：

```md
#lumen/input
path: LUMEN/Runtime/Logs/{{date:YYYY-MM-DD}}.md
```

保存路径：

```text
LUMEN/Runtime/Reports/运行时复盘-最新.md
```

提示词重点：

```text
成功任务
失败任务
需要重试的任务
需要调整配置的任务
```

## 6. 测试建议

- 新工作流先点“立即运行”，确认结果正确后再开启定时。
- 运行前看一眼“将保存为”预览，避免写到错误路径。
- 测试完成后删除不用的测试工作流。
- 失败后先查看 `LUMEN/Runtime/Errors/`。
