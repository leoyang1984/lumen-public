在回答开头必须输出这行字：「[Skill: wiki@lint 已激活]」，然后换行继续。

请对 `wiki/` 目录执行健康检查，步骤如下：

1. 调用 `search_vault` 获取 `wiki/` 下所有条目
2. 逐一调用 `read_note` 读取每篇条目
3. 检查以下问题：

   - **孤立条目**：没有任何其他条目链接到它的 `[[backlink]]`
   - **悬空链接**：条目中引用了 `[[概念名]]` 但该文件不存在
   - **内容矛盾**：不同条目中对同一概念有相互矛盾的描述
   - **缺失来源**：条目没有标注原始资料来源
   - **待补充**：内容明显不完整、有占位符或"TODO"标记
   - **未覆盖原始资料**：调用 `search_vault` 搜索 `raw/` 和 `Clippings/` 目录，找出未被任何 wiki 条目 `## 来源` 引用的文件，建议归入哪个概念

4. 输出检查报告，格式：

```
## Wiki 健康检查报告

### 孤立条目（共 N 个）
- [[条目名]] — 建议添加从 [[相关条目]] 的链接

### 悬空链接（共 N 个）
- [[条目名]] 中引用了不存在的 [[缺失概念]]

### 内容矛盾（共 N 处）
- ...

### 未覆盖原始资料（共 N 个）
- raw/文件名 — 建议归入 [[概念名]]

### 建议新增条目（共 N 个）
- 概念名 — 原因
```

如果 wiki/ 为空，告知用户先用 `/wiki@ingest` 导入原始文档。
