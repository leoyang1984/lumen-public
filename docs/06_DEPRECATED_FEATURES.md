# 06 Feature Evolution: Suspended & Archived Capabilities

As LUMEN Pro evolves towards a more specialized **Canvas-native Agentic Engine**, some legacy features from earlier versions have been strategically suspended. This guide explains why these changes were made to ensure a focused, high-performance experience.

### 🌬️ Hover AI Chat (Floating Widget)
- **Status**: Suspended.
- **What was it?**: A small floating bar that appeared whenever you selected text in the editor.
- **Why was it suspended?**:
  1. **Focus & Flow**: Floating widgets often obstruct the writing context and cause visual clutter.
  2. **Superior Alternatives**: LUMEN now prioritizes the **Sidebar Chat** for deep conversations and the **Slash Command (`/lumen`)** for quick edits. This keeps your editor clean and your "心流" (flow state) intact.

### ✂️ Web Clipper (Legacy)
- **Status**: Suspended.
- **What was it?**: A tool to scrape entire web pages into Markdown notes.
- **Why was it suspended?**:
  1. **Vault Bloat**: Full-page clipping often fills your vault with noisy, unorganized data.
  2. **Index-on-Demand**: The new **RAG (Memory Index)** and **`#lumen/research`** systems are much more efficient. Instead of "hoarding" dead pages, LUMEN now fetches and synthesizes live web data only when you actually need it.

### 📊 Sidebar Pipeline Stepper
- **Status**: Archived / Evolved.
- **What was it?**: A linear progress tracker for multi-step tasks in the sidebar.
- **Why was it archived?**:
  1. **Spatial Logic**: Linear progress bars are too restrictive for complex AI reasoning.
  2. **Visual Engine**: This feature evolved into the **Canvas Flow** system. By moving logic to the whiteboards, you can now see parallel branches, loop-backs, and complex dependencies that were impossible to represent in a sidebar list.

---

### 🛡️ Our Commitment
LUMEN's development philosophy is "Less is More, Depth is Value." We remove friction so that your Obsidian environment remains a place for deep thinking, not just another cluttered AI app.

## 中文版 {#中文版}

# 06 功能演进：停用与封存的功能说明

随着 LUMEN Pro 向更专业的**白板原生代理引擎 (Canvas-native Agentic Engine)** 演进，早期版本中的一些功能已被策略性地封存。本指南旨在解释这些变更背后的思考，以确保为您提供最专注、高性能的使用体验。

### 🌬️ 划词浮窗 AI (Hover AI Chat)
- **状态**：暂时停用 (Suspended)。
- **它是什么？**：每当您在编辑器中选中文字时，会出现一个小的浮动操作条。
- **为什么要停用？**：
  1. **专注与心流**：浮动窗口往往会遮挡笔记上下文，产生视觉干扰。
  2. **更优的替代方案**：LUMEN 现在优先强化**侧边栏对话**用于深度推演，以及**斜杠命令 (`/lumen`)** 用于快速编辑。这保证了编辑器的纯净，保护了您的“心流”状态。

### ✂️ 网页剪藏 (Legacy Web Clipper)
- **状态**：暂时停用 (Suspended)。
- **它是什么？**：将完整的网页抓取并转换为 Markdown 笔记的工具。
- **为什么要停用？**：
  1. **仓库臃肿**：全文本剪藏往往会导致笔记库中充斥着噪音数据。
  2. **按需索引**：新的 **RAG (记忆索引)** 和 **`#lumen/research`** 系统更加高效。LUMEN 不再推崇“仓鼠式”地囤积网页，而是在您真正需要时，实时抓取并总结互联网的最前沿信息。

### 📊 侧边栏流水线进度条 (Pipeline Stepper)
- **状态**：演进而关闭 (Evolved)。
- **它是什么？**：侧边栏中用于跟踪多步任务的线性进度条。
- **为什么要停用？**：
  1. **空间逻辑**：线性进度条对于复杂的 AI 推理来说过于死板。
  2. **视觉引擎化**：该功能已全面进化为 **Canvas Flow** 系统。通过将逻辑移至白板，您可以直观地看到并行分支、循环反馈以及复杂的依赖关系，这些在侧边栏列表中是无法完美呈现的。

LUMEN 的开发哲学是“少即是多，深即是值”。我们致力于消除干扰，让您的 Obsidian 环境始终是一个深度思考的场所，而非另一个臃肿的 AI 工具。
