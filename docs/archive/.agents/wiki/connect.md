在回答开头必须输出这行字：「[Skill: wiki@connect 已激活]」，然后换行继续。

请分析 `wiki/` 中各条目之间的深层关联，发现未被显式链接的潜在连接，步骤如下：

1. 调用 `search_vault` 获取 `wiki/` 下所有条目
2. 调用 `read_note` 读取各条目内容
3. 分析：

   - **隐性关联**：内容上相关但尚未互相链接的条目对
   - **共同主题**：多个条目都涉及但没有专门条目的上层概念
   - **有趣的对比**：可以放在一起对比分析的概念组合
   - **知识空白**：从现有条目推断出"应该存在但还没有"的条目

4. 输出报告：

```
## 发现的潜在连接

### 建议添加 Backlink
- [[条目A]] ↔ [[条目B]]：原因说明

### 建议新建条目
- **新概念名**：可以综合 [[条目X]] 和 [[条目Y]] 的内容写一篇

### 有趣的对比角度
- [[概念A]] vs [[概念B]]：值得对比的维度

### 值得深入探索的问题
- 问题描述（基于现有 wiki 内容自然引申）
```

**报告结束后**，询问用户是否要执行以下操作：
- 自动添加建议的 backlink（调用 `create_note` 更新对应条目）
- 跳转到 `/wiki@consolidate` 执行合并/拆分
